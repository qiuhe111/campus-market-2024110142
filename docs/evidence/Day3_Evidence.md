# Day3 证据卡：Mock 数据建模与列表渲染

## 一、我的设计

### 业务理解

**二手交易** — 核心是"闲置物品流转"，需要：
- 商品基本信息：标题、分类、价格、成色
- 交易信息：地点（校内位置）、发布者、发布时间
- 状态追踪：open / closed，方便后续扩展交易流程

**失物招领** — 核心是"信息匹配"，需要：
- 区分类型：lost（寻物）vs found（拾物）
- 物品关键特征：物品名称而非标题即可快速识别
- 地点 + 时间是匹配的关键维度
- 联系方式独立字段，不混入描述

**拼单搭子** — 核心是"凑人数"，需要：
- 人数进度：targetCount / currentCount 直观展示差额
- 截止时间：deadline 控制拼单有效期
- 类型字段区分拼餐、学习资料、出行等场景

**跑腿委托** — 核心是"任务 + 报酬"，需要：
- taskType 区分取快递、代买、占座等任务类型
- from / to 明确任务路线
- reward 用数值方便后续排序和筛选

### 数据结构设计（db.json）

每个资源 id 从 1 开始自增，字段名使用驼峰，保持与 JSON Server 默认行为一致。筛选标准：

1. 字段必须有页面可展示的对应位置（title → 卡片标题、description → 卡片描述）
2. 至少有 1 个分类/标签字段（如 category、type、taskType）用于后续筛选功能
3. 状态字段 status 保留但不强制处理，为 Day4 预留
4. 每类业务 5 条数据，覆盖不同子类型
5. 数据内容贴近真实校园生活场景

## 二、AI 设计

### 优点

1. **代码结构清晰**：按业务划分为独立 API 模块（trade.ts / lostFound.ts / groupBuy.ts / errand.ts），每个模块包含完整的 TypeScript 接口定义 + 请求函数
2. **类型安全**：为每个实体定义了完整接口（TradeItem、LostFoundItem 等），编译期类型检查避免字段拼写错误
3. **组件抽象合理**：ItemCard 组件通过 props + slot 做到通用，四类业务共用同一卡片组件，减少重复代码
4. **空状态处理**：EmptyState 组件覆盖数据为空时的 UI 展示，提升用户体验
5. **分层清晰**：Axios 基础实例（http.ts）→ 业务 API 模块（api/*.ts）→ 页面组件（views/*.vue），符合推荐的分层架构

### 问题

1. **接口只定义了 GET**：目前四个 API 模块只导出了 getXxx() 方法，缺少增删改的请求函数，后续需补充
2. **baseURL 硬编码**：http.ts 中 baseURL 写死 `http://localhost:3001`，后续接入真实后端需改为环境变量
3. **没有 loading/error 状态**：页面组件中只有数据请求成功后的赋值，缺少 loading 中/请求失败的 UI 反馈
4. **ItemCard 灵活性有限**：footer slot 虽然可自定义，但若某类业务需要更多差异化布局（如图片、按钮操作），当前卡片结构可能不够灵活

## 三、最终调整

| 条目 | 调整方向 | 说明 |
|---|---|---|
| db.json 字段设计 | ✅ 保留 | 用户自定字段，覆盖四类业务核心属性 |
| TypeScript 接口定义 | ✅ 保留 | AI 生成的接口定义与 db.json 一致 |
| API 模块拆分 | ✅ 保留 | 按业务模块划分，维护清晰 |
| Axios 实例封装 | ✅ 保留 | 基础封装，未加拦截器，保持简洁 |
| ItemCard 组件 | ✅ 保留 | 用户提供的模板代码，slot 灵活 |
| EmptyState 组件 | ✅ 保留 | 新增空状态提示 |
| baseURL 硬编码 | ⚠️ 暂不处理 | Day3 阶段不做环境变量配置，后续统一处理 |
| 缺少 CRUD 方法 | ⚠️ 暂不处理 | Day3 聚焦列表展示，增删改等 Day4 补充 |
| 缺少 loading/error | ⚠️ 暂不处理 | 保持最小实现，后续按需增强 |

### 保留的核心设计决策

- **数据驱动**：页面内容全部来自 JSON Server 接口，无任何写死数据
- **分层架构**：Axios → API 模块 → 页面组件，请求与视图分离
- **组件复用**：ItemCard 跨四类业务复用，减少重复代码
- **最小依赖**：未引入额外 UI 库、状态管理库（Pinia 已安装但 Day3 未使用）
